# Behavioral Constraints

- **Do Not Push to GitHub Directly**: Never run git commit or git push commands. Instead, make modifications to local project files, verify that everything compiles/works locally, and then provide clear, copy-pasteable terminal commands/steps for the user to run the git commands themselves.
- **Do Not Delete Original Files**: Never delete original files or media assets when optimizing or modifying resources unless explicitly requested by the user. Keep them in the workspace and track them in the repository.
